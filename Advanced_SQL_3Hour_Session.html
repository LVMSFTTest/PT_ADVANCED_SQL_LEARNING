📊 Advanced ORDER BY

Sorting goes far beyond ASC and DESC — you can sort by expressions, functions, aliases, custom logic, and handle NULLs.

Basic Syntax Recap

ORDER BY
SELECT column1, column2
FROM   table_name
ORDER BY column_to_sort [ASC | DESC];

-- Multiple columns: sort last name ASC, then first name ASC within same last name
SELECT first_name, last_name
FROM   customer_info
ORDER BY last_name ASC, first_name ASC;

Sorting with Expressions & Functions

Expressions in ORDER BY
-- Sort by name length (shortest names first)
SELECT FirstName, LENGTH(FirstName) AS name_len
FROM   customer_info
ORDER BY LENGTH(FirstName) ASC;

-- Sort by absolute value (distance from 70000)
SELECT FirstName, s.Salary,
       ABS(s.Salary - 70000) AS distance_from_70k
FROM   customer_info c
JOIN   salary s ON c.CustomerId = s.CustomerId
ORDER BY ABS(s.Salary - 70000) ASC;  -- closest to 70k first

Sorting with a Column Alias

Alias in ORDER BY
-- Use the alias 'annual_salary' in ORDER BY
SELECT FirstName, LastName,
       s.Salary * 12 AS annual_salary
FROM   customer_info c
JOIN   salary s ON c.CustomerId = s.CustomerId
ORDER BY annual_salary DESC;  -- alias works here

Sorting with CASE WHEN (Custom Priority)

🎓 Real-World Use Case

You want "New York" customers to always appear first, then everyone else alphabetically by city.

CASE in ORDER BY
SELECT FirstName, City
FROM   customer_info
ORDER BY
    CASE
        WHEN City = 'New York' THEN 0   -- New York always first
        ELSE                          1   -- everything else after
    END,
    City ASC;                             -- alphabetical within each group

Advanced Filtering — IN, AND, OR, NOT

IN

Match against a list of values. Cleaner than multiple OR conditions.

IN
SELECT * FROM customer_info
WHERE City IN ('New York', 'Miami');
NOT IN

Exclude a list of values.

NOT IN
SELECT * FROM customer_info
WHERE City NOT IN ('Los Angeles');
AND / OR / NOT
-- AND: both conditions must be true
SELECT * FROM customer_info
WHERE City = 'New York' AND Age > 30;

-- OR: at least one condition true
SELECT * FROM customer_info
WHERE City = 'New York' OR City = 'Miami';

-- NOT: negate the condition
SELECT * FROM customer_info
WHERE NOT City = 'Los Angeles';

-- Complex combination with parentheses
SELECT * FROM customer_info
WHERE (City = 'Miami' AND Age > 30)
   OR (City = 'Chicago' AND Age < 28);
💡 Use parentheses when mixing AND and OR — AND has higher precedence and can give unexpected results without them.

📊 GROUP BY, HAVING & Math

Grouping summarises rows. Math operations create computed columns. Together they power all reporting.

GROUP BY + Aggregates

🎓 Mental Model

Without GROUP BY → one number for the whole table. With GROUP BY City → one number per city.

GROUP BY
-- Customer count and average salary per city
SELECT   c.City,
         COUNT(*) AS customer_count,
         ROUND(AVG(s.Salary), 2) AS avg_salary,
         MAX(s.Salary) AS top_salary,
         SUM(s.Salary) AS total_payroll
FROM     customer_info c
JOIN     salary s ON c.CustomerId = s.CustomerId
GROUP BY c.City
ORDER BY avg_salary DESC;

-- Average salary grouped by age
SELECT   c.Age,
         ROUND(AVG(s.Salary), 2) AS avg_salary
FROM     customer_info c
JOIN     salary s ON c.CustomerId = s.CustomerId
GROUP BY c.Age
ORDER BY c.Age;

HAVING — Filter After Grouping

WHERE vs HAVING
ClauseFiltersCan use Aggregates?When it runs
WHEREIndividual rows❌ NoBefore GROUP BY
HAVINGGroups✅ YesAfter GROUP BY
HAVING
-- Cities with more than 1 customer
SELECT   City, COUNT(*) AS count
FROM     customer_info
GROUP BY City
HAVING   COUNT(*) > 1;

-- Cities where total salary > 100,000
SELECT   c.City,
         SUM(s.Salary) AS total_salary
FROM     customer_info c
JOIN     salary s ON c.CustomerId = s.CustomerId
GROUP BY c.City
HAVING   SUM(s.Salary) > 100000
ORDER BY total_salary DESC;

Mathematical Operations

Arithmetic
-- Annual salary, tax estimate, and net pay in one query
SELECT c.FirstName,
       s.Salary                         AS monthly,
       s.Salary * 12                    AS annual,
       s.Salary * 12 * 0.20             AS tax_estimate,
       s.Salary * 12 * (1 - 0.20)       AS net_annual
FROM   customer_info c
JOIN   salary s ON c.CustomerId = s.CustomerId;

-- CASE WHEN for conditional discount
SELECT FirstName, s.Salary,
       CASE
           WHEN s.Salary >= 90000 THEN s.Salary * 0.10  -- 10% bonus
           WHEN s.Salary >= 70000 THEN s.Salary * 0.05  -- 5% bonus
           ELSE                         s.Salary * 0.02  -- 2% bonus
       END AS bonus
FROM   customer_info c
JOIN   salary s ON c.CustomerId = s.CustomerId;

LIMIT with OFFSET — Pagination

LIMIT + OFFSET
-- First 5 customers ordered by last name
SELECT * FROM customer_info ORDER BY LastName ASC LIMIT 5;

-- NEXT 3 customers (rows 6-8) — skip first 5, take 3
SELECT * FROM customer_info
ORDER BY Age DESC
LIMIT 3 OFFSET 5;   -- skip 5, return rows 6-8

1. INNER JOIN — Only Matching Rows

🎓 Analogy

Only students who are enrolled AND have a grade appear. Students without a grade — excluded. Grades without a student — excluded.

INNER JOIN
-- All customers WITH a salary record (15 rows)
SELECT   c.CustomerId,
         CONCAT(c.FirstName, ' ', c.LastName) AS full_name,
         c.City,
         s.Salary
FROM     customer_info c
INNER JOIN salary s ON c.CustomerId = s.CustomerId
ORDER BY s.Salary DESC;

2. LEFT JOIN — All from Left Table

🎓 Analogy

All customers appear — even those with no salary record. Salary columns will show NULL for customers with no match.

LEFT JOIN
-- Every customer, salary is NULL if no record
SELECT   c.CustomerId, c.FirstName, c.City,
         s.Salary
FROM     customer_info c
LEFT JOIN salary s ON c.CustomerId = s.CustomerId;

-- Find customers with NO salary record (NULL filter)
SELECT   c.FirstName, c.City
FROM     customer_info c
LEFT JOIN salary s ON c.CustomerId = s.CustomerId
WHERE    s.SalaryId IS NULL;  -- customers without salary

3. RIGHT JOIN — All from Right Table

RIGHT JOIN
-- All salary records appear, even if customer is missing
SELECT   c.FirstName, c.City,
         s.SalaryId, s.Salary
FROM     customer_info c
RIGHT JOIN salary s ON c.CustomerId = s.CustomerId;

4. FULL OUTER JOIN — Everything from Both

⚡ MySQL does not support FULL OUTER JOIN directly. Simulate it using UNION of LEFT JOIN and RIGHT JOIN.
FULL OUTER (MySQL simulation)
-- Full outer join simulation in MySQL
SELECT   c.FirstName, c.City, s.Salary
FROM     customer_info c
LEFT JOIN salary s ON c.CustomerId = s.CustomerId

UNION

SELECT   c.FirstName, c.City, s.Salary
FROM     customer_info c
RIGHT JOIN salary s ON c.CustomerId = s.CustomerId;

🔄 Subqueries — Queries Inside Queries

A subquery is a SELECT statement nested inside another query. The inner query runs first. Its result is used by the outer query.

3 Types of Subqueries

Scalar

Returns a single value. Used in WHERE or SELECT comparisons.

Table Subquery

Returns rows + columns. Used in FROM clause as a virtual table.

Correlated

References the outer query. Runs once per outer row.

Scalar Subquery — Single Value

Scalar Subquery
-- Customers earning MORE than the average salary
SELECT   c.FirstName, c.City, s.Salary
FROM     customer_info c
JOIN     salary s ON c.CustomerId = s.CustomerId
WHERE    s.Salary > (
             SELECT AVG(Salary) FROM salary   -- single value: 72,600
         )
ORDER BY s.Salary DESC;

-- Find the customer with the highest salary
SELECT c.FirstName, s.Salary
FROM   customer_info c
JOIN   salary s ON c.CustomerId = s.CustomerId
WHERE  s.Salary = (SELECT MAX(Salary) FROM salary);

Table Subquery — Virtual Table in FROM

Table Subquery
-- Treat the subquery result as a temporary table
SELECT   city_stats.City,
         city_stats.avg_sal,
         city_stats.total_customers
FROM     (
             SELECT   c.City,
                      ROUND(AVG(s.Salary), 2) AS avg_sal,
                      COUNT(*) AS total_customers
             FROM     customer_info c
             JOIN     salary s ON c.CustomerId = s.CustomerId
             GROUP BY c.City
         ) AS city_stats           -- must give subquery an alias
WHERE    city_stats.avg_sal > 70000
ORDER BY city_stats.avg_sal DESC;

Correlated Subquery — Row-by-Row Comparison

🎓 How It Works

A correlated subquery references a column from the outer query. For every row the outer query processes, the inner query re-executes using that row's values.

Correlated Subquery
-- Employees earning more than their city's average
SELECT   c.FirstName, c.City, s.Salary
FROM     customer_info  c
JOIN     salary         s  ON c.CustomerId = s.CustomerId
WHERE    s.Salary > (
             SELECT AVG(s2.Salary)
             FROM   customer_info  c2
             JOIN   salary         s2 ON c2.CustomerId = s2.CustomerId
             WHERE  c2.City = c.City  -- ← references outer query's city
         )
ORDER BY c.City, s.Salary DESC;

Subquery with IN

Subquery + IN
-- Customers who live in cities that have at least 2 customers
SELECT FirstName, City
FROM   customer_info
WHERE  City IN (
           SELECT   City
           FROM     customer_info
           GROUP BY City
           HAVING   COUNT(*) >= 2
       )
ORDER BY City;

🪟 Window Functions

Window functions calculate a value for each row based on a "window" of related rows — without collapsing them into groups like GROUP BY does.

Window Functions vs GROUP BY

GROUP BY

Collapses rows into groups. You get ONE row per group.

You lose individual row data.

WINDOW FUNCTION

Keeps all rows. Adds a computed column to each row based on surrounding rows.

Individual data is preserved.

Syntax

Window Function Syntax
function_name() OVER (
    [PARTITION BY column]   -- divide rows into groups (like GROUP BY)
    [ORDER BY    column]    -- order rows within each partition
)

ROW_NUMBER, RANK, DENSE_RANK

🎓 The Difference

If two people tie for 2nd place: ROW_NUMBER gives them 2 and 3 (arbitrary tiebreak). RANK gives both 2 but skips to 4 next. DENSE_RANK gives both 2 and continues with 3.

ROW_NUMBER / RANK / DENSE_RANK
SELECT   c.FirstName, c.City, s.Salary,
         ROW_NUMBER()   OVER (ORDER BY s.Salary DESC) AS row_num,
         RANK()         OVER (ORDER BY s.Salary DESC) AS salary_rank,
         DENSE_RANK()   OVER (ORDER BY s.Salary DESC) AS dense_rank
FROM     customer_info c
JOIN     salary s ON c.CustomerId = s.CustomerId;

-- PARTITION BY: rank within each city separately
SELECT   c.FirstName, c.City, s.Salary,
         RANK() OVER (
             PARTITION BY c.City
             ORDER BY     s.Salary DESC
         ) AS city_rank
FROM     customer_info c
JOIN     salary s ON c.CustomerId = s.CustomerId
ORDER BY c.City, city_rank;

LEAD & LAG — Look Ahead / Behind

LEAD / LAG
-- Compare each salary to the next and previous salary
SELECT   c.FirstName, s.Salary,
         LAG(s.Salary)  OVER (ORDER BY s.Salary) AS prev_salary,
         LEAD(s.Salary) OVER (ORDER BY s.Salary) AS next_salary,
         s.Salary - LAG(s.Salary) OVER (ORDER BY s.Salary) AS salary_jump
FROM     customer_info c
JOIN     salary s ON c.CustomerId = s.CustomerId
ORDER BY s.Salary;

Running Total with SUM OVER

Running Total
-- Running total of salaries (cumulative, ordered by salary)
SELECT   c.FirstName, s.Salary,
         SUM(s.Salary) OVER (
             ORDER BY s.Salary
             ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
         ) AS running_total
FROM     customer_info c
JOIN     salary s ON c.CustomerId = s.CustomerId
ORDER BY s.Salary;

Get Top-N Per Group (Classic Interview Pattern)

Top-1 per city
-- Highest paid customer in each city
SELECT * FROM (
    SELECT   c.FirstName, c.City, s.Salary,
             RANK() OVER (PARTITION BY c.City ORDER BY s.Salary DESC) AS rnk
    FROM     customer_info c
    JOIN     salary s ON c.CustomerId = s.CustomerId
) ranked
WHERE rnk = 1;

🔄 ETL & Statistical Analysis

ETL (Extract, Transform, Load) is the process of moving and preparing data for analysis. SQL handles all three steps.

ETL Overview

Extract

Pull raw data

Read from CSV files, APIs, other databases using LOAD DATA or SELECT.

Transform

Clean & reshape

Remove duplicates, fix types, CASE WHEN for normalization, JOIN to combine sources.

Load

Write to target

INSERT INTO a new table or data warehouse using INSERT...SELECT.

Extract — LOAD DATA INFILE

LOAD DATA INFILE
-- Load from a CSV file into a table
LOAD DATA INFILE '/path/to/customers.csv'
INTO TABLE customer_info
FIELDS TERMINATED BY ','
ENCLOSED BY '"'
LINES TERMINATED BY '\n'
IGNORE 1 ROWS   -- skip the header row
(CustomerId, FirstName, LastName, Age, City);

-- Enable file loading if not already on
SET GLOBAL local_infile = 1;

Transform — Clean & Normalize

Transform Examples
-- Normalize salary into tiers
SELECT c.FirstName, s.Salary,
       CASE
           WHEN s.Salary / 5  <= 5  THEN s.Salary / 5.0   -- normalize 0-5 scale
           ELSE                       s.Salary / 10.0  -- normalize 0-10 scale
       END AS normalized_salary
FROM   customer_info c
JOIN   salary s ON c.CustomerId = s.CustomerId;

-- Standardize city names (trim whitespace, uppercase)
UPDATE customer_info
SET    City = TRIM(UPPER(City));

-- Fill NULL ages with the average age
UPDATE customer_info
SET    Age = (SELECT ROUND(AVG(Age)) FROM customer_info WHERE Age IS NOT NULL)
WHERE  Age IS NULL;

Load — INSERT ... SELECT

Load Step
-- Create and populate a summary table (the Load step)
CREATE TABLE city_salary_summary (
    City          VARCHAR(50),
    avg_salary    DECIMAL(10,2),
    total_salary  DECIMAL(12,2),
    customer_count INT
);

INSERT INTO city_salary_summary
SELECT   c.City,
         ROUND(AVG(s.Salary), 2),
         SUM(s.Salary),
         COUNT(*)
FROM     customer_info c
JOIN     salary s ON c.CustomerId = s.CustomerId
GROUP BY c.City;

Statistical Analysis

Variance & Standard Deviation
-- Built-in functions (MySQL supports these)
SELECT
    AVG(Salary)      AS mean,
    VARIANCE(Salary) AS variance,
    STDDEV(Salary)   AS std_deviation,
    MAX(Salary) - MIN(Salary) AS salary_range
FROM salary;

-- Manual variance calculation (for understanding)
SELECT
    SUM((Salary - AVG(Salary) OVER()) * (Salary - AVG(Salary) OVER())) /
    (COUNT(*) OVER() - 1) AS sample_variance
FROM salary
LIMIT 1;

⚙️ Stored Procedures

A stored procedure is a saved SQL routine — write it once, call it many times with different parameters.

Why Stored Procedures?

Benefits

  • Reusable — call instead of rewriting
  • Precompiled — faster execution
  • Secure — hide table structure
  • Maintainable — change logic in one place

Syntax Pattern

  • DELIMITER changes statement end marker
  • IN = input, OUT = output parameter
  • BEGIN...END wraps the logic
  • CALL runs the procedure

Basic Procedure

Simple Stored Procedure
-- Create procedure to get customer info by ID
DELIMITER //
CREATE PROCEDURE GetCustInfo (IN Customer_Id INT)
BEGIN
    SELECT * FROM customer_info
    WHERE CustomerId = Customer_Id;
END //
DELIMITER ;

-- Call it with different IDs
CALL GetCustInfo(7);
CALL GetCustInfo(12);

Procedure with Multiple Parameters

Multi-parameter Procedure
-- Get customers by city with min salary filter
DELIMITER //
CREATE PROCEDURE GetCustomersByCity (
    IN  p_city       VARCHAR(50),
    IN  p_min_salary DECIMAL(10,2)
)
BEGIN
    SELECT   c.FirstName, c.LastName, c.City, s.Salary
    FROM     customer_info c
    JOIN     salary s ON c.CustomerId = s.CustomerId
    WHERE    c.City = p_city
    AND      s.Salary >= p_min_salary
    ORDER BY s.Salary DESC;
END //
DELIMITER ;

CALL GetCustomersByCity('New York', 70000);

-- Procedure with OUT parameter
DELIMITER //
CREATE PROCEDURE GetAvgSalary (OUT avg_result DECIMAL(10,2))
BEGIN
    SELECT AVG(Salary) INTO avg_result FROM salary;
END //
DELIMITER ;

CALL GetAvgSalary(@result);
SELECT @result AS average_salary;
