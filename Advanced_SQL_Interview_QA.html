ORDER BY — Advanced Sorting
8 questions
1
What does ORDER BY do and what is the default sort order?
▾
Answer
ORDER BY sorts the result set of a query. The default is ASC (ascending) — smallest to largest, A to Z, oldest to newest. You must explicitly write DESC for descending order. Without ORDER BY, the database can return rows in any order — never assume a default order.
SELECT FirstName, LastName FROM customer_info ORDER BY LastName ASC; SELECT FirstName, Salary FROM customer_info ORDER BY Salary DESC;
2
How do you sort by multiple columns?
▾
Answer
List the columns separated by commas. Sorting is applied left to right — the first column is the primary sort key, and the second column is used only when two rows have the same value in the first column.
-- Sort by city A-Z, then by salary highest-first within each city SELECT FirstName, City, Salary FROM customer_info c JOIN salary s ON c.CustomerId = s.CustomerId ORDER BY City ASC, Salary DESC;
3
Can you use a column alias in ORDER BY? Can you use it in WHERE?
▾
Answer
Yes to ORDER BY, No to WHERE. ORDER BY runs after SELECT, so aliases are already defined by the time ORDER BY executes. WHERE runs before SELECT, so the alias doesn't exist yet.
SELECT FirstName, Salary * 12 AS annual FROM customer_info c JOIN salary s ON c.CustomerId = s.CustomerId ORDER BY annual DESC; -- ✅ works -- WHERE annual > 800000 ← ❌ fails — alias not known yet
4
How do you sort by a function or expression?
▾
Answer
You can put any valid SQL expression directly in the ORDER BY clause — including function calls like LENGTH(), ABS(), or arithmetic.
-- Sort customers by name length (shortest first) SELECT FirstName FROM customer_info ORDER BY LENGTH(FirstName) ASC; -- Sort by absolute distance from age 30 SELECT FirstName, Age FROM customer_info ORDER BY ABS(Age - 30) ASC;
5
How do you create a custom sort order using CASE WHEN in ORDER BY?
▾
Answer
Place a CASE WHEN expression directly inside ORDER BY. Assign numeric priorities — lower numbers sort first. This lets you control exactly which values appear at the top.
-- New York customers appear first, then everyone else by city SELECT FirstName, City FROM customer_info ORDER BY CASE WHEN City = 'New York' THEN 0 ELSE 1 END, City ASC;
6
What is LIMIT and what does OFFSET do?
▾
Answer
LIMIT n returns only the first n rows. OFFSET n skips the first n rows before returning results. Together they implement pagination — skip page 1, return page 2.
SELECT * FROM customer_info ORDER BY Age DESC LIMIT 5; -- rows 1-5 SELECT * FROM customer_info ORDER BY Age DESC LIMIT 3 OFFSET 5; -- rows 6-8
7
How do you find the 2nd highest salary using ORDER BY?
▾
Answer
Use ORDER BY DESC to rank highest first, DISTINCT to avoid ties fooling the rank, LIMIT 1 to take one row, and OFFSET 1 to skip the top one.
SELECT DISTINCT Salary AS second_highest FROM salary ORDER BY Salary DESC LIMIT 1 OFFSET 1;
8
Why should you always use ORDER BY with LIMIT?
▾
Answer
Without ORDER BY, the database can return rows in any order — which changes between runs. LIMIT without ORDER BY gives you unpredictable, non-deterministic results. Always pair them.
⚠️ SELECT * FROM customer_info LIMIT 5 — this can return different rows each time. Always add ORDER BY to get consistent results.
Advanced Filtering — IN, AND, OR, NOT
8 questions
1
What does the IN operator do and why is it better than multiple OR conditions?
▾
Answer
IN checks whether a column value matches any value in a provided list. It is functionally identical to chaining OR conditions but is far more readable and maintainable — especially with large lists.
-- Verbose with OR WHERE City = 'Chicago' OR City = 'Houston' OR City = 'Seattle' -- Clean with IN WHERE City IN ('Chicago', 'Houston', 'Seattle')
2
What is the difference between AND and OR operators?
▾
Answer
AND requires both conditions to be true — it narrows results (fewer rows). OR requires at least one condition to be true — it broadens results (more rows).
-- AND: only customers in NY AND older than 30 WHERE City = 'New York' AND Age > 30 -- OR: customers in NY OR Miami (either city counts) WHERE City = 'New York' OR City = 'Miami'
3
What is operator precedence between AND and OR? How do you control it?
▾
Answer
AND has higher precedence than OR — it is evaluated first, just like × before + in arithmetic. Use parentheses to control the order explicitly and avoid bugs.
-- Without parentheses: AND runs first, then OR WHERE City = 'Miami' AND Age > 30 OR City = 'Chicago' -- Reads as: (Miami AND age>30) OR Chicago -- With parentheses: explicit and clear WHERE (City = 'Miami' AND Age > 30) OR (City = 'Chicago' AND Age < 28)
4
How does NOT work and when would you use NOT IN?
▾
Answer
NOT negates a condition — it returns rows where the condition is false. NOT IN excludes a list of values.
-- Exclude Los Angeles customers SELECT * FROM customer_info WHERE NOT City = 'Los Angeles'; -- or: WHERE City != 'Los Angeles' -- NOT IN excludes multiple values WHERE City NOT IN ('Los Angeles', 'Seattle')
⚠️ NOT IN with NULLs returns 0 rows — if the list contains NULL, SQL cannot confirm "not equal to NULL". Use IS NOT NULL separately.
5
What is BETWEEN and is it inclusive?
▾
Answer
BETWEEN filters values within a range and is fully inclusive on both ends — the start and end values are included in the results.
-- Both 25 and 35 are included SELECT * FROM customer_info WHERE Age BETWEEN 25 AND 35; -- Equivalent: WHERE Age >= 25 AND Age <= 35
6
How do you find rows where a column contains a specific substring?
▾
Answer
Use LIKE with % wildcards on both sides. The % matches any sequence of characters, so '%a%' finds any value containing 'a' anywhere in the string.
-- First names containing the letter 'a' SELECT * FROM customer_info WHERE FirstName LIKE '%a%'; -- Last names starting with 'D' SELECT * FROM customer_info WHERE LastName LIKE 'D%';
7
How do you filter rows where a column is NULL or NOT NULL?
▾
Answer
Use IS NULL or IS NOT NULL. You cannot use = NULL — that always returns no rows because NULL is not equal to anything, not even itself.
-- Wrong — always returns 0 rows WHERE Salary = NULL -- Correct WHERE Salary IS NULL WHERE Salary IS NOT NULL
8
How do you combine LIKE with AND to find names of a specific length?
▾
Answer
Combine LIKE with LENGTH() in a WHERE clause. LIKE handles the pattern (e.g. starts with a character) and LENGTH() enforces the exact character count.
-- First name starts with 'J' AND is exactly 4 characters SELECT * FROM customer_info WHERE FirstName LIKE 'J___' -- J + 3 wildcards = 4 chars AND LENGTH(FirstName) = 4; -- redundant here but clear
GROUP BY, HAVING & Math Operations
8 questions
1
What does GROUP BY do and what rules must you follow when using it?
▾
Answer
GROUP BY divides rows into groups based on shared column values and applies aggregate functions per group. Rule: every column in SELECT that is NOT inside an aggregate function MUST appear in GROUP BY.
-- ✅ Correct: City is in both SELECT and GROUP BY SELECT City, COUNT(*) AS count FROM customer_info GROUP BY City; -- ❌ Wrong: FirstName not in GROUP BY or aggregate -- SELECT FirstName, City, COUNT(*) FROM customer_info GROUP BY City;
2
What is the difference between WHERE and HAVING?
▾
Answer
WHERE filters individual rows before grouping and cannot use aggregate functions. HAVING filters groups after GROUP BY and can use aggregate functions like COUNT, AVG, SUM.
-- Customers in cities with more than 2 people, earning > 50k SELECT City, COUNT(*) AS cnt, AVG(Salary) AS avg_sal FROM customer_info c JOIN salary s ON c.CustomerId = s.CustomerId WHERE s.Salary > 50000 -- row filter (before grouping) GROUP BY City HAVING COUNT(*) > 2; -- group filter (after grouping)
3
How do you group by multiple columns?
▾
Answer
List multiple columns in GROUP BY — one group is created for each unique combination of values across all listed columns.
-- One row per unique (City, Age) combination SELECT City, Age, COUNT(*) AS count FROM customer_info GROUP BY City, Age ORDER BY City, Age;
4
How do you calculate a running total in SQL?
▾
Answer
Use SUM() as a window function with OVER() and ORDER BY. This adds a cumulative total to each row rather than collapsing all rows into one.
SELECT FirstName, Salary, SUM(Salary) OVER (ORDER BY Salary) AS running_total FROM customer_info c JOIN salary s ON c.CustomerId = s.CustomerId;
5
How do you use CASE WHEN inside an aggregate function?
▾
Answer
Wrap a CASE WHEN inside an aggregate function to conditionally count or sum only rows that match a condition — a technique called conditional aggregation.
-- Count high and low earners per city in one query SELECT City, COUNT(CASE WHEN s.Salary >= 70000 THEN 1 END) AS high_earners, COUNT(CASE WHEN s.Salary < 70000 THEN 1 END) AS low_earners FROM customer_info c JOIN salary s ON c.CustomerId = s.CustomerId GROUP BY City;
6
What arithmetic operators does SQL support?
▾
Answer
SQL supports + (add), - (subtract), * (multiply), / (divide), % or MOD (remainder). These can be used in SELECT, WHERE, HAVING, and ORDER BY clauses.
SELECT FirstName, Salary, Salary * 12 AS annual, Salary * 12 * 0.20 AS tax, Salary MOD 1000 AS remainder FROM customer_info c JOIN salary s ON c.CustomerId = s.CustomerId;
7
How do you find the total salary paid per city, only for cities over $100,000?
▾
Answer
SELECT c.City, SUM(s.Salary) AS total_salary FROM customer_info c JOIN salary s ON c.CustomerId = s.CustomerId GROUP BY c.City HAVING SUM(s.Salary) > 100000 ORDER BY total_salary DESC;
8
How do you find duplicate records in a table?
▾
Answer
GROUP BY the columns that define uniqueness, then use HAVING COUNT(*) > 1 to find groups with more than one row — those are duplicates.
SELECT FirstName, LastName, City, COUNT(*) AS cnt FROM customer_info GROUP BY FirstName, LastName, City HAVING COUNT(*) > 1;
JOINs — All 4 Types
8 questions
1
What is the difference between INNER JOIN, LEFT JOIN, RIGHT JOIN, and FULL OUTER JOIN?
▾
Answer
INNER JOIN: only rows matching in both tables. LEFT JOIN: all rows from the left table, NULLs for unmatched right. RIGHT JOIN: all rows from the right table, NULLs for unmatched left. FULL OUTER JOIN: all rows from both, NULLs wherever there's no match.
SELECT c.FirstName, s.Salary FROM customer_info c INNER JOIN salary s ON c.CustomerId = s.CustomerId; -- 15 rows (all match) LEFT JOIN salary s ON c.CustomerId = s.CustomerId; -- all customers, NULL if no salary
2
How do you find records that exist in one table but NOT in another?
▾
Answer
Use a LEFT JOIN and then filter WHERE the right table's key IS NULL. This returns all rows from the left that have no match in the right table.
-- Customers who have NO salary record SELECT c.FirstName, c.City FROM customer_info c LEFT JOIN salary s ON c.CustomerId = s.CustomerId WHERE s.SalaryId IS NULL;
3
Does MySQL support FULL OUTER JOIN? How do you simulate it?
▾
Answer
MySQL does NOT support FULL OUTER JOIN syntax. Simulate it using UNION of a LEFT JOIN and a RIGHT JOIN — UNION automatically removes duplicates.
SELECT c.FirstName, s.Salary FROM customer_info c LEFT JOIN salary s ON c.CustomerId = s.CustomerId UNION SELECT c.FirstName, s.Salary FROM customer_info c RIGHT JOIN salary s ON c.CustomerId = s.CustomerId;
4
What is a self join and when would you use it?
▾
Answer
A self join joins a table to itself using aliases. Use it when rows in the same table are related to each other — e.g. employees and their managers stored in the same table.
-- Find customers older than another customer in the same city SELECT a.FirstName, a.Age, b.FirstName AS younger_in_city FROM customer_info a JOIN customer_info b ON a.City = b.City AND a.Age > b.Age;
5
What is the difference between UNION and UNION ALL?
▾
Answer
UNION removes duplicate rows from the combined result. UNION ALL keeps all rows including duplicates. UNION ALL is faster because it skips the deduplication step.
SELECT City FROM customer_info WHERE Age > 35 UNION -- removes duplicates SELECT City FROM customer_info WHERE Age < 25; SELECT City FROM customer_info WHERE Age > 35 UNION ALL -- keeps all, including duplicates SELECT City FROM customer_info WHERE Age < 25;
6
What happens to rows when using INNER JOIN if one table has no matching row?
▾
Answer
With INNER JOIN, rows with no match in either table are completely excluded from the result. If customer_info has a customer with no salary record, that customer will not appear at all in an INNER JOIN with the salary table.
⚠️ If you expect all rows from one table to appear even without a match, use LEFT JOIN instead of INNER JOIN.
7
How do you join three or more tables?
▾
Answer
Chain multiple JOIN clauses. Each JOIN adds one more table. The ON clause specifies the relationship between that table and any previously joined table.
-- If we had a third table: customer_info → salary → bonus SELECT c.FirstName, s.Salary, b.BonusAmount FROM customer_info c JOIN salary s ON c.CustomerId = s.CustomerId JOIN bonus b ON s.SalaryId = b.SalaryId;
8
What is the ON clause in a JOIN? Can you JOIN on more than one condition?
▾
Answer
The ON clause specifies which columns to match between the two tables. You can join on multiple conditions by chaining them with AND.
-- Join on a single condition ON c.CustomerId = s.CustomerId -- Join on multiple conditions ON c.CustomerId = s.CustomerId AND c.City = 'New York'
Subqueries — Nested Queries
8 questions
1
What is a subquery? What are the 3 types?
▾
Answer
A subquery is a SELECT statement nested inside another query. The inner query runs first. Three types: Scalar (returns one value, used in WHERE/SELECT), Table subquery (returns rows/columns, used in FROM), Correlated (references the outer query, runs per outer row).
2
How do you find customers earning above the overall average salary?
▾
Answer
-- Scalar subquery in WHERE SELECT c.FirstName, c.City, s.Salary FROM customer_info c JOIN salary s ON c.CustomerId = s.CustomerId WHERE s.Salary > (SELECT AVG(Salary) FROM salary) ORDER BY s.Salary DESC;
3
What is a correlated subquery? Why is it slower than a regular subquery?
▾
Answer
A correlated subquery references a column from the outer query — it cannot run independently. It re-executes once for every row the outer query processes. For a table with 10,000 rows, the subquery runs 10,000 times — making it potentially slow for large datasets.
-- Runs the inner SELECT once per outer row SELECT c.FirstName, c.City, s.Salary FROM customer_info c JOIN salary s ON c.CustomerId = s.CustomerId WHERE s.Salary > ( SELECT AVG(s2.Salary) FROM customer_info c2 JOIN salary s2 ON c2.CustomerId = s2.CustomerId WHERE c2.City = c.City -- ← references outer query );
4
How do you use a subquery in the FROM clause (table subquery)?
▾
Answer
Put the subquery inside parentheses in the FROM clause and give it an alias — it behaves as a virtual table. This is also called a derived table or inline view.
SELECT stats.City, stats.avg_sal FROM ( SELECT c.City, ROUND(AVG(s.Salary), 2) AS avg_sal FROM customer_info c JOIN salary s ON c.CustomerId = s.CustomerId GROUP BY c.City ) AS stats -- alias is required WHERE stats.avg_sal > 70000;
5
What is the difference between a subquery and a CTE (WITH clause)?
▾
Answer
Both achieve similar results. A CTE is named and defined once at the top using WITH...AS — making complex queries far more readable and reusable within the same query. A subquery is inline and anonymous. CTEs can also be recursive.
-- Subquery version (harder to read) SELECT * FROM (SELECT City, AVG(Salary) FROM ...) x WHERE x.avg > 70000; -- CTE version (much cleaner) WITH city_avg AS ( SELECT City, AVG(Salary) AS avg FROM ... ) SELECT * FROM city_avg WHERE avg > 70000;
6
How do you use EXISTS vs IN for subqueries?
▾
Answer
IN compares a value against a result set — it materialises the full list. EXISTS returns true as soon as it finds the first match — it is often faster for large datasets because it short-circuits.
-- IN: customers whose ID exists in salary table SELECT * FROM customer_info WHERE CustomerId IN (SELECT CustomerId FROM salary); -- EXISTS: faster for large tables SELECT * FROM customer_info c WHERE EXISTS ( SELECT 1 FROM salary s WHERE s.CustomerId = c.CustomerId );
7
How do you write a subquery inside an UPDATE statement?
▾
Answer
Use a subquery in the SET clause or the WHERE clause of an UPDATE to calculate or filter dynamically.
-- Set salary to the company average for anyone below it UPDATE salary SET Salary = (SELECT AVG(Salary) FROM ( SELECT Salary FROM salary ) AS tmp) WHERE Salary < (SELECT AVG(Salary) FROM ( SELECT Salary FROM salary ) AS tmp2);
8
How do you find customers living in cities that have at least 2 customers (using a subquery)?
▾
Answer
SELECT FirstName, City FROM customer_info WHERE City IN ( SELECT City FROM customer_info GROUP BY City HAVING COUNT(*) >= 2 ) ORDER BY City;
Window Functions
8 questions
1
What is a window function and how is it different from GROUP BY?
▾
Answer
A window function computes a value for each row based on a "window" of related rows. Unlike GROUP BY, it does NOT collapse rows — every row remains in the output, with an extra computed column added. Uses OVER() clause.
-- GROUP BY: 6 rows (one per city) SELECT City, AVG(Salary) FROM ... GROUP BY City; -- Window: 15 rows (all customers + avg column) SELECT FirstName, City, Salary, AVG(Salary) OVER (PARTITION BY City) AS city_avg FROM ...;
2
What is the difference between RANK(), DENSE_RANK(), and ROW_NUMBER()?
▾
Answer
Given a tie for 2nd place between two rows: ROW_NUMBER assigns 2 and 3 (arbitrary tiebreak). RANK assigns 2 to both but skips 3, giving 4 next. DENSE_RANK assigns 2 to both and continues with 3 — no skipping.
SELECT FirstName, Salary, ROW_NUMBER() OVER (ORDER BY Salary DESC) AS rn, RANK() OVER (ORDER BY Salary DESC) AS rnk, DENSE_RANK() OVER (ORDER BY Salary DESC) AS drnk FROM customer_info c JOIN salary s ON c.CustomerId = s.CustomerId;
3
What does PARTITION BY do in a window function?
▾
Answer
PARTITION BY divides the data into groups (like GROUP BY) but the window function resets and recalculates for each partition independently. All rows still appear in the output.
-- Rank salary within each city separately SELECT FirstName, City, Salary, RANK() OVER ( PARTITION BY City ORDER BY Salary DESC ) AS city_rank FROM customer_info c JOIN salary s ON c.CustomerId = s.CustomerId;
4
How do you get the top-1 earner per city using window functions?
▾
Answer
Rank salaries within each city using RANK() with PARTITION BY, wrap it in a subquery, then filter WHERE rank = 1. This is one of the most common interview patterns.
SELECT * FROM ( SELECT c.FirstName, c.City, s.Salary, RANK() OVER (PARTITION BY c.City ORDER BY s.Salary DESC) AS rnk FROM customer_info c JOIN salary s ON c.CustomerId = s.CustomerId ) ranked WHERE rnk = 1;
5
What do LEAD() and LAG() do?
▾
Answer
LAG(col) returns the value of a column from the previous row. LEAD(col) returns the value from the next row. Both operate within the window defined by OVER. Useful for period-over-period comparisons.
SELECT FirstName, Salary, LAG(Salary) OVER (ORDER BY Salary) AS prev_salary, LEAD(Salary) OVER (ORDER BY Salary) AS next_salary FROM customer_info c JOIN salary s ON c.CustomerId = s.CustomerId;
6
How do you calculate a cumulative (running) total using window functions?
▾
Answer
SELECT FirstName, Salary, SUM(Salary) OVER ( ORDER BY Salary ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW ) AS cumulative_salary FROM customer_info c JOIN salary s ON c.CustomerId = s.CustomerId ORDER BY Salary;
7
What is NTILE() and what is it used for?
▾
Answer
NTILE(n) divides rows into n equal buckets and assigns a bucket number (1 to n) to each row. Commonly used for quartiles (NTILE(4)) or percentiles (NTILE(100)).
-- Divide customers into salary quartiles (4 equal groups) SELECT FirstName, Salary, NTILE(4) OVER (ORDER BY Salary) AS salary_quartile FROM customer_info c JOIN salary s ON c.CustomerId = s.CustomerId;
8
Can you use a window function in a WHERE clause?
▾
Answer
No — window functions cannot appear in WHERE or HAVING directly because they are evaluated after those clauses. Wrap the window function in a subquery or CTE, then filter in the outer query.
-- Wrong: cannot use RANK() in WHERE -- WHERE RANK() OVER (...) = 1 ← Error! -- Correct: wrap in subquery first SELECT * FROM ( SELECT *, RANK() OVER (PARTITION BY City ORDER BY Salary DESC) AS rnk FROM customer_info c JOIN salary s ON c.CustomerId = s.CustomerId ) x WHERE rnk = 1;
ETL, Stored Procedures & Statistics
7 questions
1
What does ETL stand for and what does each step do in SQL?
▾
Answer
Extract: pull data from source (LOAD DATA INFILE, SELECT). Transform: clean and reshape — deduplicate, fix types, normalize with CASE WHEN, join sources. Load: write to target — INSERT...SELECT into a summary or warehouse table.
2
How do you load data from a CSV file into a MySQL table?
▾
Answer
LOAD DATA INFILE '/path/to/file.csv' INTO TABLE customer_info FIELDS TERMINATED BY ',' ENCLOSED BY '"' LINES TERMINATED BY '\n' IGNORE 1 ROWS -- skip header row (CustomerId, FirstName, LastName, Age, City);
💡 Enable file loading first: SET GLOBAL local_infile = 1;
3
How do you use INSERT...SELECT to copy data into a new table?
▾
Answer
INSERT...SELECT combines creating and populating a table in one step. The SELECT can include JOINs, GROUP BY, and transformations — the result is inserted into the target table.
CREATE TABLE city_summary (City VARCHAR(50), avg_salary DECIMAL(10,2), total INT); INSERT INTO city_summary SELECT c.City, ROUND(AVG(s.Salary), 2), COUNT(*) FROM customer_info c JOIN salary s ON c.CustomerId = s.CustomerId GROUP BY c.City;
4
What is a stored procedure and what are its 4 key benefits?
▾
Answer
A stored procedure is a named, reusable block of SQL saved on the server. Benefits: Modularity (reusable blocks), Performance (precompiled = faster), Security (hide table structure, control access), Maintainability (change logic in one place).
5
Why do you need DELIMITER when creating a stored procedure?
▾
Answer
MySQL uses semicolon ; to detect the end of each statement. A stored procedure body contains semicolons inside it. Changing the DELIMITER to // tells MySQL to wait for // before executing — preventing it from ending the procedure early on the first internal semicolon.
DELIMITER // -- change delimiter to // CREATE PROCEDURE GetCustInfo (IN id INT) BEGIN SELECT * FROM customer_info WHERE CustomerId = id; END // -- procedure ends here DELIMITER ; -- reset to normal CALL GetCustInfo(7);
6
What are IN, OUT, and INOUT parameters in a stored procedure?
▾
Answer
IN: value passed into the procedure (read-only inside). OUT: value the procedure writes out to the caller (the result). INOUT: passed in, can be modified, and the modified value is returned.
DELIMITER // CREATE PROCEDURE GetAvg (OUT result DECIMAL(10,2)) BEGIN SELECT AVG(Salary) INTO result FROM salary; END // DELIMITER ; CALL GetAvg(@avg); SELECT @avg;
7
How do you calculate variance and standard deviation in MySQL?
▾
Answer
MySQL has built-in functions VARIANCE() and STDDEV(). You can also calculate them manually using SUM and AVG for deeper understanding.
-- Built-in functions (easiest) SELECT AVG(Salary) AS mean, VARIANCE(Salary) AS variance, STDDEV(Salary) AS std_deviation, MAX(Salary) - MIN(Salary) AS range_val FROM salary;
